Near Line Batch Ticket

includes the ability to generate a near line batch ticket. A near line batch ticket is defined as an additional spread to an existing batch component but will not be included "in-line" with the press job generated. Near line batch tickets will typically be sent to a laser jet style printer, not to a press. Additionally, near line batch tickets will not display on the press jobs tab within and will not be included in job related calculations and counts.

Utilizing near line batch tickets is accomplished via adding an additional spread to the desired batch component. There are two methods that can be used for device selection when working with near line batch tickets.

  • Specify a DFE and device
  • Specify a device group and device group mode
Specify a DFE and device

This method allows the administrator to set a specific DFE and destination device for the near line batch ticket to be sent. An example of this configuration method is shown below.

The use of the inline="false" attribute tells that this additional spread should not be included "inline" with the job that is going to press, but instead should go to the DFE and device specified within this additional spread declaration.

 

Nearline Specific XML
<!-- Nearline Batch Ticket Sent To Specific DFE-->
<spread inline="false" name="nearlinetoCAL-STD-1" layout="A4-LANDSCAPE" dfe="ULTRA-3-6-1" device="CAL-STD-1" relatedcontenturi="YB_idf_batchticket_8X10" relatedcontenttype="IDFBatchTicketV2"/>
 

In this example, the following will occur:

  • The batch ticket will not be included in the press PDF for this batch component
  • The batch ticket will be sent to a device called CAL-STD-1 via the Ultra-3-6-1 DFE
 Specify a device group and device group mode

This method allows the administrator to call on a specific device group for submitting the near line batch ticket. The following example illustrates how to configure an additional spread for submitting a near line batch ticket via a specific device group.

Device Group Specific XML
<!-- Nearline Batch Ticket Sent To Device Group Calendar -->
<spread inline="false" name="nearlinetoDeviceGroupCalendar" layout="A4-LANDSCAPE" deviceGroup="calendar" deviceGroupMode="remake" relatedcontenturi="YB_idf_batchticket_8X10" relatedcontenttype="IDFBatchTicketV2"/>
 

Setting up a near line batch ticket

The following steps illustrate the requirements for setting up a near line batch ticket.

  1. Identify the batch component that the near line batch ticket will be associated with
  2. Add an additional spread to this batch component (use the examples from above as a guide)
  3. Ensure the additional spread includes the inline="false" attribute
  4. Determine if the near line batch ticket will be sent to a specific DFE and device or to a device group and modify the additional spread accordingly
  5. Specify an appropriate name for the additional spread
  6. Specify the appropriate batch ticket relatedcontenturi and batch ticket type relatedcontenttype
  7. If using device group selection, specify the appropriate deviceGroupMode (standard, remake, priority)
  8. Save the batch definition

Remember that near line batch tickets do not show up on the press jobs tab in . If there is a need to inspect the PDF after the batch ticket is printed, you must browse to the location where the PDF was written to and view it manually.
