System configuration

To use a SmartStacker integration, the following directory structures must be configured within your environment.

Create the finishing directories

  1. Browse to the sharedconfig location on the database server for the environment you are working in
  2. Create a new folder called finishing at the root of sharedconfig
  3. Open the finishing folder and create a new folder called laycrimplayout inside of finishing
  4. Create a new jdf file within the laycrimplayout folder with a name that corresponds to the product you are working with. For example, if you are integrating Smart Stacker to work with a new 5 x 7 Card product, you might name the file cardA5x7.laycrimplayout.jdf

Create the blanks folder

The imposition folder in sharedconfig will contain a folder named blanks. This folder will hold the PDFs used for generating blanks and printed waste in the batched output. There will need to be a set of blanks and printed waste PDFs (front and back) that match the final trim size of each product that will use a SmartStacker finishing plan. For example, if the final trim size of our card product will be 5 x 7, then we will need to create 4 single page PDF files. Two of the files will be for the Added Blanks front and back, and the remaining two will be for the Printed Waste front and back.

  1. Browse to sharedconfig\imposition on the database server for the environment you are working in
  2. Create a new folder called blanks
  3. Create a set of PDFs for the product you are working with and copy them to the blanks folder created in step 2
