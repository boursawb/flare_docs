Automatic file clean-up

Once the event message has been processed by the system consuming the message, it should be removed from the location where it was written to by . Following integration best practices, the system that consumed the message should handle any necessary clean-up of the files. In the event this cannot occur, includes a facility to perform the necessary clean-up.

Configuration

To configure one or more clean-up locations, follow the steps shown below.

  1. Browse to the sharedconfig location
  2. Locate the util folder, create it if it doesn't exist
  3. Open the util folder and locate the cleanup folder, create it if it doesn't exist
  4. Open the cleanup folder and create a new file called eventCleanUp.cleanup.xml (or any name you prefer)
  5. Paste in the contents from the below example as a model to work from
  6. Make any necessary changes, save and close the file such as path, poll interval, or delete older than
  7. Create any additional clean-up files required

<Cleanup>
	<Path>\\localhost\idf\test</Path>
	<PollInterval>8h</PollInterval>
	<DeleteOlderThan>48h</DeleteOlderThan>
	<Extensions>
		<Ext>csv</Ext>
		<Ext>xml</Ext>
		<Ext>txt</Ext>
	</Extensions>
</Cleanup>

You can define as many extensions as required in order to specify the types of files that should be removed from this location. A seperate *.cleanup.xml file will need to be created for each unique location where should check.
