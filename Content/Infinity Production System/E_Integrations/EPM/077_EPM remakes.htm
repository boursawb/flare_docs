EPM remakes

When a remake batch is generated for one or more EPM enabled orders, the EPM disposition specified for the remake batch must match the disposition of the original orders being remade. This will ensure that the remake items match the same color gamut as other items in the original order, previously printed. Remake orders do not get evaluated for EPM compatibility through the normal workflow. For a remake order to evaluate as EPM, the order must inject with EPM_Eval with a value of true and EPM_EvalResultRemakeOverride dynamic attribute, with a value matching one of the following.

  • EPM
  • EPM_BW

Given that items contained within a remake batch are controlled by the remake tray ID, it is important that only orders that have the same value for EPM exist in the same remake batch. If a remake batch is created that results in a mix of EPM dispositions across the orders contained, the batch will fail.
