Content sized batch ticket

In most cases, a batch ticket will be sized to fit the entire spread, regardless of the signature size of the output being generated. However, in some cases, it may be desirable to constrain the size of the batch ticket to the size of the signatures being imposed. This can be useful for solutions which automatically cut the output but still require a batch ticket to be included on the stack. Content sized batch tickets can be configured to arrive on the top or bottom of the stack, as required by finishing processes.

Including a content sized batch ticket

To add a content sized batch ticket, add the following XML to the desired batch component requiring the batch ticket.

<AdditionalContent>
	<Content>
		<ContentType>IDFBatchTicketV2</ContentType>
		<ContentUri>IDBookBatchTicket</ContentUri>
		<Duplex>true</Duplex>
		<ReverseOrder>true</ReverseOrder>
		<Width>860</Width>
		<Height>612</Height>
		<InsertFirst>false</InsertFirst>
	</Content>
</AdditionalContent>

Additional content elements can be included within the additional content. For example, it may be desired to include 1 batch ticket at the front of the stack and 1 at the batch. Or, possibly include more than one batch ticket at the end of the job.

ContentType - Defines the type of related content being used

ContentUri - Defines the unique identifier of a file

Duplex - If true, will pad to an even number of pages

ReverseOrder - If true, will add the file to the imposition in reverse order

Width - Width of the component in points. This must match the width of the actual output

Height - Height of the component in points. This must match the height of the actual output

InsertFirst - If true, the batch ticket will be inserted before the content specified in the prebatch components order section of the batch component. If false, the batch ticket will be inserted at the end. An example of a content sized batch ticket is shown below.
