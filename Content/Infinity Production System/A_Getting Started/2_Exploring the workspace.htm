Exploring the workspace

The user interface consists of multiple tabs. Each of these tabs represent a portion of the work flow as orders move through the system. As you become more familiar with IPS, think of the tabs like they are on a hill, with the products tab at the top of the hill and press jobs being at the bottom. This analogy will help you understand why certain things happen in the order they do.

Products Tab

The products tab is the main view for displaying orders currently in the system. The products tab allows you to search, view, filter, and sort orders to see only those you are interested in.

The context menu of the Products tab (right-click menu), consists of the following actions and categories. Expand the category to see the options available.

Root level
  • Create Batch Create a batch from the currently selected order(s)
  • Create Single Product Batch Provides a way to select multiple orders and then create a unique batch for each order in a single action
  • Delete Order Delete the currently selected order(s)
  • Mark Order Highest Priority Change the priority of the selected order from the default of 50 to 100
  • Divert to alternate output profile Divert the work to an alternate output profile for reprocessing
  • Clear Filters Clear all filters that have been applied, making all rows will be visible
 Reprocess
  • Reprocess at Order Analyze Reprocess an order, sending it back to the analyze phase of the work flow
  • Reprocess at Order Prebatch Reprocess an order, sending it back to the prebatch phase of the work flow
  • Reprocess at Order VDP Merge Reprocess an order, sending it back to the VDP merge phase of the work flow
  • Reprocess at Order Sort Reprocess an order, sending it back to the sort phase of the work flow
 Copy
  • Copy Error Information For an order that is in an error state, highlighting this row and selecting this option from the menu will copy the error information to the Windows clipboard
  • Copy Cell Contents Right-click on a value in a cell and select this option to copy the value to the Windows clipboard
View
  • View History Opens the order history dialogue for reviewing the status trail of the order
  • View Prebatch Opens each of the generated prebatch components within the default PDF viewer application
  • View Order Attributes Opens a web page that displays all of the attributes currently in memory, associated with the selected order
  • View Order Attributes from DB Opens a web page that displays all of the attributes defined in the database for the selected order
  • View Order Content Files Displays a Windows explorer dialogue containing the original order content as they were received from the customer
Designer
  • Archive Test Order Exports the currently selected order(s) to a chosen location. Archived orders are typically used for testing and validation
  • Mark Order For Designer Mode Sets a flag on the order that results in decaf content being used, with each order designated with a random color
  • Un-mark Order For Designer Mode Sets the order back to using its original order content
  • Product Configuration Browser Launches the product configuration browser that is used for viewing and managing product configuration files
  • Export Slate Designer Data Export an XML document that is used for generating reports within Jaspersoft Studio
  • Export Selected Table Data Exports the currently selected rows to a tab deliniated file
 Proofs
  • Mark Order Requires Proof Sets a flag on the order signifying that a proof should be generated
  • Create Proof Creates a proof order of with a single quantity for validation purposes

The products tab is typically used for reviewing order information or troubleshooting issues that may arise. For generating batches and releasing work to press, this will most often be handled from the sort groups tab.

Sort Groups Tab

When it comes to the daily operations of most production environments, the vast majority of your time will be spent on the sort groups tab. The sort groups tab provides several options for batching work to press as well as providing a very informative view for managing your current workload. Sort profiles can be used to dynamically adjusts how batches are built to accommodate rapidly changing business needs. Additionally, you have the ability to enable auto-release with the desired auto-release filter for quickly and efficiently releasing work to press.

By right-clicking on a sort group in the UI, the following actions can be performed.

Root level
  • Reprocess at Order Reprocess all of the orders contained in the currently selected sort group as the sort phase of the work flow
  • Release Many Batches release a specific number of batches or to release batches older than a specified date
  • Split Sort Group Splits the currently selected sort group(s) by based on the selected criteria
  • Un-split Sort Group / Merge to Parent Merges the selected sort group(s) back to the parent it was split from
  • Manual Release Next Batch Releases the next batch in line from the sort group by presenting the batch dialogue to the user so the appropriate destination device can be selected
  • Release Next Batch Releases the next batch in the sort group to the appropriate device group
  • Manual Release All Releases all of the batches existing in the sort group by presenting the batch dialogue to the user so the appropriate destination device can be selected
  • Release All Releases all batches from the sort group to the appropriate device group
  • Divert to alternate output profile Displays the divert to alternate output profile dialogue which allows the user to divert all work in the currently selected sort group to an alternate output profile
  • Clear Filters Clear all filters that have been applied, ensuring that all rows will be visible
  • View Sort Group History Opens the sort group history dialogue for reviewing events specific to the currently selected sort group
  • View Sort Group Error Displays a dialogue showing the details of the error on the sort group
  • View Sort Group Attributes Displays a web page showing all attributes associated with the currently selected sort group
  • Export Selected Table Data Export the currently selected rows to a tab deliniated file
  • Copy Cell Contents Right-click on a particular attribute in a cell and copy that information to the Windows clipboard

The Batches Tab

The batches tab displays all current batches in the system. Each row on the batches tab can be expanded to display the orders contained within that batch. From this screen you can view attributes associated with the batch, review orders contained within the batch, and reprint one or more batches, amongst other activities.

By right-clicking on a batch in the UI, the following actions can be performed.

Root level

View Batch History Opens the batch history dialogue for reviewing the status trail of the batch

View Batch History Performs a reprint on the orders within the selected batch and displays the batch dialogue so the appropriate destination device can be selected

Reprint Batch Performs a reprint of the orders in the batch and displays the batch dialogue so the appropriate destination device can be selected

Reprint Batch Single Order per Batch Reprints each order in the current batch to a new batch and displays the batch dialogue so the appropriate destination device can be selected

Divert Batch To Alternate Output Displays the batch divert dialogue in order to divert the batch to the desired output profile

Delete Batch Deletes the currently selected batch(es)

View Batch Attributes Displays a web page showing all attributes associated with the currently selected batch

Copy Error Information Copies the error information associated with the batch to the Windows clipboard

Clear Filters Clears the currently applied filters to ensure that all rows are visible

Export Selected Table Data Exports the currently selected rows to a tab deliniated file

Copy Cell Contents Allows you to right-click on a particular attribute in a cell and copy that information to the Windows clipboard

The Press Jobs Tab

The press jobs tab shows detailed information about each job submitted to press, or a defined none location. Jobs submitted to devices that are fully integrated with IPS will show their status as they move through press workflow. The press jobs tab also provides the ability to reprint a batch. This can be very useful when an exception batch needs only 1 or 2 of the press jobs reprinted instead of the entire set.

By right-clicking on a press job in the UI, the following actions can be performed.

Root level

Manual Reprint Reprint orders within the selected batch and displays the batch dialogue so the appropriate destination device can be selected

Reprint Batch reprint of the orders in the batch and automatically sends the output to the appropriate destination device

View Press Job PDF Opens the selected press job in the default PDF viewer

Copy Error Information Copies the error information associated with the press job to the Windows clipboard

Delete Deletes the currently selected press job(s)

Clear Filters Clears the currently applied filters to ensure that all rows are visible

Export Batch Ticket Designer Data Exports an XML file which provides a datasource for creating custom batch tickets in iReport

View Press Job Attributes Displays a web page showing all attributes associated with the currently selected press job

Export Selected Table Data Exports the currently selected rows to a tab delineated file

Copy Cell Contents Copies the data from the cell that was right-clicked on to the clipbord

The Post Press Tab

The Post Press tab provides detailed information on jobs which have been configured for a post press workflow.

The Administration Tab

The administration tab displays information about order injection statistics, injection history and active cluster member information. From here you can see when the last injection took place, view injection history and display that information on a graph. In the event errors occur during order injection, the administration tab will contain a link to the folder containing the errors (for orders that injected via a hotfolder).

The Cluster Admin section is broken down in to two parts, cluster members and active tasks. The cluster members section displays a summary of all distributed cores, order injectors and UIs that are active across all servers and workstations within the group. The active tasks section shows the active processes within the cluster, which host they are running from, and the last time the process has executed
