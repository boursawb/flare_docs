Auto-release filters

When using auto-release, a filter can be applied which limits what can release automatically. This is accomplished by using a matcher to identify a subset of orders that can be released while the filter is in play. This means that filters are only limited by the criteria that can be defined in a matcher. In most auto-release enabled environments, the majority of sort groups will be configured to auto-release. For example, if you had to increase the amount of work going to press for a specific customer, you could create a filter profile which only allows orders with a customer name of PrintMark to be released automatically. Until the filter is turned off, all other products would have to be manually released by an operator.

If there is a need to change the filter that is being used, auto-release must be disabled and turned on again to select a new filter profile.

Creating an auto-release filter

An auto-release filter is comprised of two components, a matcher and an autorelease profile. The matcher is no different than one you would create to sort orders. Release filters reside in the sharedconfig\ordersort\autoreleaseprofile folder on the master core host server. The naming convention for the auto-release filter must be in the form of name.autoreleaseprofile.xml. An example of a properly named auto-release filter would be PrintMark.autoreleaseprofile.xml.

Create the matcher
  1. Launch the Sort Rule Configuration tool
  2. Within the Matcher Actions section, click the Create a new matcher link
  3. Specify the desired name for the match (for example - AF-PrintMark)
  4. Select Order or Product from the first drop-down list
  5. Select the desired attribute to match against (for example - Customer)
  6. Select the appropriate operand in the Is drop-down list
  7. Specify a value to match against (for example - PrintMark)
  8. Click Ok to save the new matcher
 Create the auto-release filter
  1. Make a connection to the master core host server
  2. Open the sharedconfig\ordersort\autoreleaseprofile location
  3. Edit one of the existing auto-release filters using Notepad++ or other text / XML editor
  4. Change the value of the description attribute to match the new filter
  5. Change the value of the matcher attribute to the name of the previously created matcher
  6. Perform a Save As to save the filter with the new name

Example XML

Auto-release Filter XML

<autoreleaseprofile description="PrintMark Customer" matcher="AF-PrintMark">
</autoreleaseprofile>
 Setting an auto-release filter
  1. Change to the sort groups tab in the UI
  2. Click the button to turn on auto-release
  3. Select the radio button to turn on auto-release
  4. Select the radio button to use a filter
  5. Select the desired filter from the drop-down list
  6. Click Ok
