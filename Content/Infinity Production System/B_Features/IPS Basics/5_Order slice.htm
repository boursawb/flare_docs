Order slices

For high volume production environments, uses order slices to segment the total orders in the system into smaller sets based on criteria you define. If custom slices have not been built, the system will automatically generate a slice for each set of 100,000 orders that exist. By using order slices, it is possible to have several hundred thousand orders in the system concurrently, but still view them in smaller more manageable chunks known as a slice. Both system generated and user defined order slices can be viewed by clicking the button in the upper left corner of the Products tab in the UI, as shown below.

When selecting a specific slice, it only impacts the orders displayed in the UI where the slice was selected. For example, if workstation A selects the ABC-Co., it has not impact on workstation B that is currently using the All system generated slice.

Switching between slices is not meant to be a quick or frequent task. When dealing with slices containing upwards of 100,000 orders, switching from one slice to another can take several minutes. The goal of a slice is to section off a large set of orders so they can be managed independently, isolated from all other orders in the system

Configuring an order slice

Each order slice will consist of the following 5 parameters.

  • Matcher A matcher will be used to identify the orders that should be included in the slice.
  • Image An image can be suplied to more easily identify the slice on the chooser dialog.
  • Page size The page size specifies the maximum number of orders that can exist in the slice before a new slice will be generated. If page size has been set to 50,000, a new slice will be generated for the next order received matching the defined criteria.
  • System generated This is a boolean flag which identifies the slice as either system generated or user defined. All custom built slices should have this value set to false.
  • Display order The display order specifies the order of the slices as they are shown on the chooser dialog.

All order slices must reside in the following location on the master core host (DB server).

\\<yourServer>\idf\sharedconfig\ordersort\orderslice

Creating an order slice

  1. Identify the criteria that will be used to assign orders to the slice
  2. Use the Sort Rule Configuration Tool to build the new matcher, or create the matcher manually via the XML configuration files directly
  3. Create a new XML file in the order slice directory mentioned above (in sharedconfig location). Make sure the file name follows the convention <yourSliceName>.orderslice.xml
  4. Copy the contents of an existing system generated order slice into the new order slice XML created in the previous step.
  5. Update each of the 5 attributes in the new order slice to the appropriate values
  6. Save the new order slice
  7. Click the order slice button to ensure the new slice displays on the slice chooser dialog

Once the slice has been saved, click the order slice button at the top of the products tab in the UI and verify the new slice displays as expected.

To select a slice, simply click the button below the icon for the order slice to load. If there is no need to change the current slice, click the Ok button to close the dialog. If a slice has been selected that displays a subset of the total orders in the system, a**truncated resultsmessage will be displayed next to the order count in the upper left corner of the UI (products tab).
