UI Perspectives

A UI perspective, often referred to as a UI profile allows you to configure the user interface to suit a specific need and then save the state of the UI with a chosen name that can be recalled as needed for future use. Most configurable items within the UI can be saved to a perspective.

  • Column position
  • Column size
  • Sorting of columns
  • Filtered columns
  • Containing specific search criteria

It is important to understand that when you save a perspective, you are saving the state of all tabs within the UI, not just the one you are on. As you make changes to each of the tabs, save the perspective often to ensure your changes are not lost. UI perspectives make it very easy for users with different job responsibilities to create their own UI experience. Once built, UI perspectives will be available to all users, regardless of who created them, or which workstation they were created from.

Example

It may be desirable to have a perspective which filters the UI by a category of Reprint/Remake for work that has be submitted to press again. Only showing orders that meet this criteria could be accomplished by using UI perspectives.

UI perspectives can be accessed from three locations within the UI.

  • Tools menu
  • Drop-down list
  • Function keys

The tools menu provides the main interface for managing existing and creating new perspectives as well as the ability to select the currently active perspective. The drop-down selection box in the lower right portion of the UI allows you to select an existing perspective, as well as refresh the currently selected perspective. Refreshing the current perspective can be used to return the UI to the initial state of the perspective in the event any changes were made after it was initially loaded. When accessing perspectives from the tools menu, the following options are available.

Tools menu options

Load Perspective Allows you to select an existing perspective and make it active.

Reset Perspective Is used to restore the UI configuration to the point in time when the perspective first loaded. This can be used to reverse unwanted changes that have been made following the initial load of the perspective.

Save Perspective Saves changes to the current perspective. You can also use this option to save a new perspective.

Remove Perspective Allows the user to remove a perspective from the system.

View UI Perspective History Displays the history dialogue for the active perspective.

Working with perspectives

When working with perspectives, the following actions can be performed.

Load UI perspective

The following steps illustrate how to load a perspective.

  1. Click Tools from the main menu
  2. Select UI Perspectives
  3. Click Load Perspective
  4. Select the desired perspective from the drop-down box
  5. Click Ok
 Reset UI perspective

The following steps illustrate how to reset the current perspective.

  1. Click Tools from the main menu
  2. Select UI Perspectives
  3. Click Reset Perspective

Alternatively, you can use the key combination CTRL+U, or click the refresh button (blue arrows) next to the perspective quick menu in the status bar.

 Save UI perspective

The following steps illustrate how to save changes to the current perspective.

  1. Click Tools
  2. Select UI Perspectives
  3. Click Save a Perspective
  4. On the Save UI Perspective dialogue, select one of the following
    1. Save as new perspective, providing a new name for the perspective
    2. Overwrite existing perspective, selecting the perspective from the drop-down to overwrite
  5. Click OK
 Modify UI perspective

The following steps illustrate how to modify an existing perspective.

  1. Make adjustments to column positioning, sorting and filtering as needed. Be sure to make changes on ALL tabs.
  2. Follow the steps above for saving changes to perspectives
 Remove UI perspective

The following steps illustrate how to remove (delete) an existing perspective.

  1. Click Tools
  2. Select UI Perspectives
  3. Click Remove a perspective
  4. Select the perspective to be removed from the drop-down list
  5. Click OK
 View UI perspective history

The following steps illustrate how to view UI perspective history

  1. Click Tools
  2. Select UI Perspectives
  3. Click View UI perspective history
 Managing UI perspectives

UI perspectives reside in the ui\uiprofile sub-folder of sharedconfig on the master core host server. Each perspective built results in a single XML file that can be migrated to other systems for use. For example, a perspective that has been created in a stage environment can be copied to the production environment or even to a different site, as required. The following steps illustrate how to migrate perspectives from one location to another.

  1. Browse to the sharedconfig folder on the master core host and open the UI\uiprofile sub-folder
  2. Copy the UI perspective(s) to be implemented
  3. Paste them in to the same location within the environment that needs them
