Working with orders

You can interact with an order by simply clicking on the row and highlighting it in the UI. Once the order is highlighted, there are several actions that can be performed using the context (right-click) menu. For detailed information about the available actions that can be performed, see Exploring the workspace

Searching for orders

The search field in the upper right corner of the UI provides a way to enter a string of text to find orders containing that value. When searching for orders, it is important to understand that the column including the value you are looking for must be enabled and visible, or the order will not be displayed in the results found.

Search for orders
  1. Identify the field that will be used for locating the order (customer, order id, etc.)
  2. Ensure this column is currently enabled on the products tab
  3. Type or paste in the value of the item being searched for

The UI will automatically filter down to only show rows that contain the value entered.

Searching for deleted order history

Once an order has been deleted, IPS retains history about that order for 7 days. Even though the order itself is not visible within the UI, it is still possible to retrieve the history for an order has been removed. This can be invaluable for troubleshooting that may need to occur after an order was deleted from the system. The following steps can be performed to retrieve history on an order that has been removed from the system within 7 days of the delete occurring.

  1. Right-click on any order within the UI
  2. Select the option to View Order History
  3. On the order history window, clear the value currently in the text field labeled Order History Window
  4. Type in the order ID in question
  5. Click the blue refresh icon
  6. Review the history

If no history is displayed, the order ID entered was not correct / complete or the order was deleted greater than 7 days from the time of the search.

Performing actions on orders

When working with orders, there are several actions that can be performed. To perform any of the following actions on one or multiple orders, right click on the highlighted order(s) and then select the desired action.

Mark Order Highest Priority

Marking an order with high priority (100) will result in moving the order to the top of the queue for purposes of batching. When batches are released, they are initially ordered as defined by the sort rules in play. If a particular order currently exists in the 6th batch of a given sort group but needs to be printed immediately, the order can be marked with high priority to move it to the next batch to be released.

Reprocess an order

Orders can be reprocessed at various stages of the workflow. The 4 stages where a reprocess can be performed are shown below in the order of execution.

  • Analyze
  • VDP Merge
  • Prebatch
  • Sort

The VDP step is only visible when variable data orders exist in the system. Additionally, there is a fifth state/light which displays in the UI for injection, but the ability to reprocess at this step does not exist and is accomplished by resubmitting the order from the order management system.

It is important to understand that when an order is reprocessed, it will move through all steps of the workflow from the point of execution. If an order is reprocessed at analyze, it will again go through the VDP, prebatch and sort steps. Conversely, if an order is reprocessed at the end of the workflow, at sort for example, it will reprocess at the sort step only. Understanding this concept is critical as it can result in duplicate work being sent to press. For example, if an operator had reprocessed an order after it had already been successfully batched and printed, the order would return to the sort groups and be available for batching a second time, ultimately resulting in duplicate work. Some of the issues that can result in needing to reprocess orders include the following.

  • Configuration files for a given product have changed since the initial injection of the order
  • External resources were not available during the initial processing of the order resulting in a workflow step failure
  • System resource issues resulting in a workflow step failure
  • Product configuration files do not exist for a given product type

In general, if an order fails at some point in the workflow, you will need to reprocess the order at the failed step or before. To reprocess one or multiple orders, follow the steps shown below.

  1. Highlight the order(s) that need to be reprocessed
  2. Right-click and select the appropriate reprocess option
 Divert to alternate output profile

Diverting orders to an alternate profile essentially takes the order and reprocesses it at the prebatch step of the workflow using a different set of product configuration files than had previously been used. For example, this can be very useful when orders originally arrived prepared for a sheet fed press but then need to be redirected to a roll fed press. The concept of diverting work essentially allows you to switch how an order will be processed on the fly.

Divert order to an alternate profile
  1. Right-click on the order(s) that should be diverted
  2. Select the option to divert to an alternate output profile
  3. Select the desired output profile from the drop-down on the output profile dialogue box
  4. Click OK
  5. Wait for the orders to complete reprocessing before performing any further actions

View Order History

View order history allows you to see what steps of the workflow an order has progressed through as well as the details of each step. Order history can be used for determining when certain actions took place, if a particular action occurred more than one time, which servers or workstation was responsible for a given action as well as provide detailed information related to errors that may have occurred. When selecting the option to view order history, a dialogue similar to the following will display.

The left window pane of the history window shows the events and the time each event occurred. When selecting an event on the left, the details for the associated event display on the right. In the illustration shown above, View Order History has been selected and we can see the details of the event pane. By default, events are listed in descending order by their time stamp. Selecting any row from the event properties window and pressing CTRL+C will copy the details of that event to the Windows clipboard. Alternatively, you can click the scissors icon in the upper right corner to copy the event properties detail to the clipboard. To view the information, click the magnify glass icon to show the information in the default web browser.

 View PreBatch

Prebatch is the initial set of output generated by IPS that will be fed in to the batching process, ultimately resulting in the job that is sent to press. Think of prebatch as the initial preparation that is performed on the content received for the order. There are several functions that can occur at the prebatch step of the workflow including, sizing of content, splitting of content, adding additional content such as custom slates and banners, calculations performed against quantity, as well as several others. Selecting the option to view prebatch will open each PDF that has been generated during the prebatch phase of the workflow..

If an application for viewing PDF files has not been installed on the system, attempting to view prebatch will result in no error or PDF being displayed. Additionally, if disk utilization has exceeded the system threshold for generating prebatch, you will not be able to view prebatch. When a batch has successfully printed, all associated prebatch output will be deleted, preventing prebatch from being viewed.

 Create Batch

Create batch allows you to select one or more items from the products tab and send them to a specific press. Batches of orders can be manually constructed via sorting and filtering on the products tab by selecting the desired orders, right-clicking them, and then selecting the option to Create Batch. Typically, this feature is used for getting a specific order to press to meet a given deadline or having to reprint an item due to problems in finishing. When performing a Create Batch from the products tab, you will be presented with a dialogue showing the order batch(es) that will be sent to press. You can then select the desired press for the job to be sent to and click OK.

Archive Test Order

This feature allows the user to export the content and associated order XML to a specified location. Archiving an order is typically used for troubleshooting problems with an order or archiving it off as a backup. To archive an order, follow the steps shown below.

  1. Select the order(s) from the products tab to be archived
  2. Right-click on the selection and select Archive Test Order
  3. Select the location to archive the orders to from the Save dialogue box
 Delete Order

Delete order is used to remove orders no longer needed from the system. When deleting orders, it is important to understand that orders are at the top of the delete hierarchy and cascades down. This means that when orders are deleted from the system, they are removed from the associated batch as well. Additionally, if all orders contained within a batch are deleted, the batch will also be removed. The removal of the batch will in-turn delete all press jobs associated with the batch.

Given that the delete function cascades downward, the following rules apply.

 View Order Attributes

This feature allows the operator to display all attributes associated with a given order in a web browser. Some of the more common attributes include customer, order ID, and product type. Given the number of attributes typically associated with an order, it is not realistic to include all of them as available columns within the UI. However, there are many situations where viewing an attribute not currently displayed in the UI may be necessary. There are two options for viewing order attributes.

  • View order attributes
  • View order attributes from database

View order attributes will display all attributes currently in memory for the selected order. View order attributes from database will display all attributes for the selected order, both in-memory as well as those in the database.

Copy Error Information

This feature allows you to highlight a row that is in an error state, select the option to copy error information and have this information sent to the Windows clipboard. This is very useful for collecting preliminary information about a problem that can be sent to our support team for review.

Clear Filters

Clear filters is used to remove all filters that are currently applied in the UI. This should be the first thing to try when unable to locate orders that are believed to be in the system.

 Product Configuration Browser

The product configuration browser provides a graphical view of the IPS product templates. Product configuration browser is typically used during the template development process and for template modifications when troubleshooting issues. When launching the product configuration browser, it will default to display information for the product currently selected in the UI. Depending on the number of products that have been defined in the system, it can take a few moments to launch. Some of features available within the product configuration browser include:

  • Viewing the structure of product templates
  • Exporting product configuration files
  • Template editing
  • Locating a given template in its containing folder
  • Launching the imposition viewer tool
  • Searching for existing products

When launching the configuration browser for the first time during a UI session, you may receive a message indicating that one or more products failed to load. If you choose to see the errors, additional prompts will occur showing each product template that failed to load. You will then have to acknowledge each failing product before the configuration browser will successfully load.

Imposition Preview Tool

The imposition preview tool is typically used by a product template developer for viewing how the signatures will be imposed on the sheet. This tool provides the following capabilities to the developer.

  • A graphical view of how the product will be imposed based on the imposition selected
  • Ability to change various page attributes, sheet count and zoom
  • Ability to select different impositions and see the results in real-time
  • Ability to add additional impositions (compound impositions)
  • Creation of decaf PDF files for testing

 Export Slate Designer Data

 View Order Content Files

This feature allows you to view the original order content files as they were received from the order management system. Depending upon the type of order selected (PDF/JPG), will either launch the associated PDFs in the default PDF viewer or open the location containing the JPG files.

Copy Cell Contents

This captures the value displayed in any cell within the UI. The information will be sent to the Windows clipboard where it can be pasted in to other fields within the UI or other applications as needed.

UI data export

The data export feature allows you to export a tab delineated set of data from any tab within the UI to a .tsv file.

  1. Select the tab containing the data to export
  2. Right-click on any row
  3. Select the Export table data option
  4. Browse to the location to save the .tsv to
  5. Enter a name for the new file
  6. Click the Open button to save the file
