Defining the data format

The format of the data exported from the post-press solution is defined using the template found in the sharedconfig\vdp\dataexport folder. This template defines the columns of data that will exist in the exported CSV file. Each column will include a header and an expression that uses the velocity syntax for performing calculations on the values as required. An example template used for a card / envelope product where the post-press data was submitted to an envelope inserter is shown below.

Example 
<vdpexport>
	<delimiterChar>,</delimiterChar>
	<columns>
		<!-- The exclamation point after the dollar sign means 'suppress 
		the variable if null'. This is nice if you don't want $ output -->
		<column>
			<header>LineItemID</header>
			<expression>$!data.LineItemID</expression>
		</column>
		<column>
			<header>StatusDateTime</header>
			<expression>$!DateFormatUtils.format($date, 
			"yyyy/dd/MM HH:mm:ss")</expression>
		</column>
		<column>
			<header>Order Number</header>
			<expression>$!data.get("Order Number")</expression>
		</column>
		<column>
			<header>Barcode Info</header>
			<expression>$!data.LineItemID*$!data.OLDistID*$formatter.format
			('%06d', $data.DistLinNumber)</expression>
		</column>
		<column>
			<header>SKU</header>
			<expression>$!data.SKU</expression>
		</column>
		<column>
			<header>SenCompanyName</header>
			<expression>$!data.SenCompanyName</expression>
		</column>
		<column>
			<header>SenNamePrefix</header>
			<expression>$!data.SenNamePrefix</expression>
		</column>
		<column>
			<header>SenFirstName</header>
			<expression>$!data.SenFirstName</expression>
		</column>
		<column>
			<header>SenLastName</header>
			<expression>$!data.SenLastName</expression>
		</column>
….continue with more columns, then end the xml tags
	</columns>
</vdpexport>

The only supported format for the data export at this time is CSV.
