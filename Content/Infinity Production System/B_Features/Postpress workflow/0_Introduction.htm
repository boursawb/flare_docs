Post-press

The post press workflow provides a method for to communicate with downstream devices once the job has been batched and successfully printed. Press jobs that are configured to run through a post press workflow will automatically show up on the Post Press tab in the UI.

A post-press workflow provides a way for to collect information about an item being printed and then pass that information on to a down-stream finishing device. One common post-press solution would be to collect data about a press job for a 5x7 card product and then export that information to a CSV file that will ultimately be consumed by an envelope inserter machine.

All data collected during the post-press process will be pulled from a press job. The data that is extracted from the press job is determined by the VDP layout file you implement. The extraction of these attributes from the press job will result in a comma separated value (CSV) file that can then be passed on to, or consumed by a down-stream finishing device. The following steps provide an overview of what will be required to implement a post-press workflow within your environment.

  1. Identify a product and associated component to be used for the post-press workflow
  2. Configure the system to identify the above mentioned press-jobs
  3. Configure down-stream finishing equipment to utilize the exported data file
