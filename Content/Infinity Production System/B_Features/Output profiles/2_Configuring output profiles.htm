Configuring output profiles

Before orders can be diverted, must be configured to utilize output profiles.

Update sharedconfig

  1. Connect to the master core host server (database server)
  2. Open the sharedconfig location
  3. If one does not exist, create a new folder called outputprofile
  4. Create a sub-folder within outputprofile called outputprofiledef
  5. Create a new XML file within the outputprofiledef sub-folder

    6. The name given to the output profile will be the name you see when performing a divert action. Make sure each output profile created has an extension of .outputprofiledef.xml. For example - myProfile.outputprofiledef.xml

  6. Edit the newly created profile and add the following contents to it

    7. <outputprofile></outputprofile>

  7. Close and save the file
