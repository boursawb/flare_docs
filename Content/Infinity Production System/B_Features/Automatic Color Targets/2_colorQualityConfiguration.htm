Color quality configuration

Before starting the configuration process, it is important to understand a key concept of this color quality system known as target thresholds. includes two types of thresholds defined in each product bin.

  • Quantity threshold
  • Light volume threshold

A quantity threshold sets the number of spreads that must accumulate for a given counter before a color target will automatically be sent to the device that exceeded the threshold. For Counter per Press configurations, there will be a counter created for each unique occurrence of a given product bin and press. Each counter must exceed the defined threshold before a color target will be triggered. For All Press configurations, every spread printed, regardless of the device the job was printed on will result in the counter incrementing for the appropriate product bin. The second type of threshold is the light volume threshold and relies on the start of shift indicator to function. The light volume feature will submit a color target to a press if a quantity trigger has not occurred at least one time within the period specified for light volume threshold. In order for a light volume trigger to fire, the following events must occur.

  • The Start of Shift button is clicked at the beginning of the shift
  • Jobs are submitted which increments the counter but the total spreads printed does not exceed the defined quantity threshold
  • The amount of time from the start of shift to the current time has exceeded the value specified for the light volume threshold
  • At least one spread must be printed after the light volume expiration, still not exceeding the defined quantity threshold

Following these 4 events, a color target will be submitted to the press associated with this counter.

If at any point the target threshold is exceeded, resulting in a target being submitted to press, the light volume threshold for that counter is disabled until the next start of shift is indicated

Color quality Web Administrator

To configure the color quality system using the Infinity Admin web interface, the following three tasks must be completed.

  • Create one or more product bins
  • Configure the barcode settings
  • Create the color target

With the exception of creating a new color target, all Color Quality configurations will be performed using the web UI.

Create product bin(s)
  1. Launch the admin tool by clicking the Configure button on the Color Quality tab
  2. Click Product Bins button
  3. Click the blue floating action button to add a new product bin
  4. Specify a value for Product Bin Name
  5. Specify a value for Code

  6. Select a color targe from the Color Target drop-down

    If a new color target is required, refer to the following section for steps on creating a new target. This can be changed at a later time, so using the default color target that is included in as an interim solution is fine.

  7. Specify a value for Target Threshold Spreads
  8. Specify a value for Light Volume Threshold Time i.e. (1h, 2h, etc.)
  9. Check the box Create for Counter Per Press if creating a Counter per Press product bin
  10. Click the Apply button
  11. Click the Batch Components tab at the top of the product bin configuration screen
  12. Select the desired batch definition from the drop-down list
  13. Select the desired batch component from the second drop-down list
  14. Click the Add button to add the batch definition / component combination to the product bin
  15. Repeat this process for each item to be added to the product bin
  16. Click Apply

Barcode configuration parameters

The barcode settings section allows you to specify the site and press information that will be included in the barcode stamp on the color target, if used. The barcode configuration is broken down in to three sections; site information, press numbers and press types. The required parameters for each of these sections along with definitions for each are shown below.

Site Information

The site information section provides human readable as well as numeric information which identifies the site and project information specific to your organization. The information contained within the site section is applicable to all product bins and color targets and cannot be configured uniquely at the product bin level.

  • Location Code Text A unique name which identifies the location
  • Location Code Number A unique number which can be used in the color target barcode that identifies the location
  • Project Code Text A unique name which identifies the project
  • Project Code Number A unique number which can be used in the color target barcode that identifies the project

If you are unsure what values to use for project and location code, using an abbreviated version of the company name for project and abbreviated version of your site location for location should be sufficient. For example, for a Portland office of Infinity Digital, we may use INF for the project and PDX for the location.

Understanding press numbers

Each value specified for a Press Name in the Press Numbers section must match the name of a device defined in the DFE configuration file. You can determine the values which can be used by viewing the information available in the DFE and Devices tool located in the tools menu.

Identifying press names using the DFE and Devices Tool
  1. Launch DFE and Devices from the tools menu in the UI
  2. Filter the DFE column by the DFE containing the devices of interest
  3. Note the press names in the Device Name column that you wish to use and enter them in to the press numbers section of the barcode settings

The following image shows 3 presses associated with the VM-ULTRACORE1 DFE. The values listed in the Device Name column must match the value specified for the Press Name parameter defined in the Press Numbers section.

Press Numbers

  • Press Name The name of the press that will receive color targets
  • Press Number A unique numeric value for the given press

Each entry on the Press Numbers tab must have a unique press number.

Understanding press types

For each press (device) that will receive color targets, a press type will need to be defined. For HP Production Pro implementations, press type is provided by the digital front end and does not rely on the information contained within the color quality configuration. For HP Ultracore implementations, press type is not provided by the DFE and will require the information be specified in the Press Type section. Each device entry within the Ultracore DFE configuration file must include a modelname attribute that identifies the press type associated with the given device.

Example DFE config (Ultra implementation)

The value specified for model name in the DFE configuration file must match one of the values specified for Press Type within the Press Types section of the barcode settings configuration.

Additional press types can be added as new devices with differing model names are added to the DFE configuration.

Press Types

  • Press Type The name of the press type that will be used for automated color targets
  • Press Type Number A unique numeric value for the given press type

Like the press numbers section, each press type defined must have a unique press number for the given press type. Four additional parameters exist which are not configurable via the web admin UI. These include the amount of time in the past the color quality system will look for printed jobs, the interval at which this check occurs, the location where the barcode XML will be written to, and a peak season flag for enabling / disabling the Infinity logo on triggered color targets. The default values for these settings will be suitable for most implementations. If there is a need to change these parameters, contact Infinity Digital Support for further assistance.

Configure barcode settings
  1. Click Barcode Settings in the navigation menu of the Admin UI
  2. Select the Site Information tab
  3. Specify the location code text to be used
  4. Specify the location code number to be used
  5. Specify the project code text
  6. Specify the project code number
  7. Click the apply button
  8. Click the press numbers tab
  9. Type the press name in the text box beneath Press Name
  10. Type a unique number in the text box beneath Press Number
  11. Click the green Add button
  12. Click the Apply button
  13. Click the Press Types tab
  14. Type the press type name in the Press Type text box
  15. Type a unique number in the Press Type Number text box
  16. Click the green Add button
  17. Click the Apply button

Create the color target

A color target is comprised of a TIF graphics file, a color target description file (.xml) and a layout file which converts the TIF image to a PDF that can be submitted to press. The layout can also be used to include human readable text as well as image stamps and bar codes that can be used at key points throughout the production workflow.

Color target file names

Each of these files must follow a very specific naming pattern. Not following this pattern will result in the system failing to find the defined color target. It is also important to note that color target file names, including extensions, are case sensitive. The format and convention that must be used is shown below.

  • Color Target <yourName>.colortarget.tif
  • Color Target Description <yourName>.colortarget.xml
  • Color Target Layout <yourName>.template.xml

Create a color target

Color target configuration

  1. Identify a TIF graphic that will be used for the color target
  2. Use a file name for the TIF image that identifies the color target being created. This must be in the form of <yourName>.colortarget.tif
  3. Create a color target description file which matches the name of the TIF file. For example, if the name of the TIF file was infQualityTarget.colortarget.tif then we would create a color target description file called infQualityTarget.colortarget.xml:

The <yourName> portion of the color target graphic file and the color target description file must match exactly. For example: infQuality.colortarget.tif and infQuality.colortarget.xml.

Color target description contents

As the color target description may not applicable to your integration of the Infinity color quality solution, it is a required component for automated color targets to function as expected. If using the standard implementation (no custom integrations with third party color qualification systems), simply copy the contents shown below in to your color target description xml file.

<com.infdig.framework.qcs.colortarget.ColorTarget>
 <colorTargetId>infQualityTarget</colorTargetId>
 <targetPath>auto</targetPath>
 <layout>auto</layout>
 <numberOfChartColumns>21</numberOfChartColumns>
 <numberOfChartRows>27</numberOfChartRows>
 <verticalPatchSize>13.5</verticalPatchSize>
 <horizontalPatchSize>13.5</horizontalPatchSize>
 <maxwellTargetIdNumber>00000001</maxwellTargetIdNumber>
 <colorShuttle>MX*</colorShuttle>
 <mdRecordId>MD01*</mdRecordId>
</com.infdig.framework.qcs.colortarget.ColorTarget>
  1. Set the colorTargetId field within the color target description file to the same name given for the <yourName> portion of the color target graphic file.
  2. Save and close the color target description file
  3. Place the <yourName>.colortarget.tif and <yourName>.colortarget.xml files previously created in sharedconfig\qcs\static\colortargets
  4. Create a new layout file which matches the <yourName> portion of the color target that includes the .template.xml extension. For example, using our target name from above, we would create a new templated layout file called infQualityTarget.template.xml

Color target press settings

At times, there may be a need to include specific press settings along with the color target job submission. This is accomplished in the same manner that is done for standard product job submission. Color targets do not have a specific batch definition associated with them, so the following actions can be taken to ensure a specific set of press settings are used when submitting color targets to press.

Configure press settings
  1. Copy an existing batch definition to a new file and name it CQPressSettings.xml (or any preferred name)
  2. Edit the batch definition and change the value for ID to CQPS (color quality press settings)
  3. Modify the press settings section of the batch definition to include the required values (psOp section)
  4. Save the new batch definition
  5. Add this batch definition ID and one of the batch component names to the top of the product batch components section of the product bin xml file
  6. Save the product bin

It does not matter which batch component is used for this process. The key is to have the top entry in the product bin for batch components be the one which contains the press settings to use for color target submission.
