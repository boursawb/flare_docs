Color quality system

includes a color quality feature that will automatically submit a color target to press when a defined threshold has been exceeded. This is accomplished by tracking the number of spreads printed for all of the product components contained within a given product bin. A product bin is a collection of products that has been logically grouped together based on customer requirements. Each product bin is configured with a threshold that determines when an automated color target will be sent to press. When submitting a target to press, will submit the job with the highest priority and send it to the proof tray to be made available ahead of other work currently on press. also provides a way for administrators to manually send a color target to a specific press. A manual target is submitted by selecting the appropriate product bin and press from the Color Quality tab within and clicking the Generate Color Target button. Additionally, each product bin can be configured with a light volume threshold which monitors how much time has passed since the start of shift and submits a target to press if the threshold is exceeded prior to a target threshold occurring.

The Infinity Color Quality system is currently integrated and fully supported with the HP Production Pro and HP Ultra Digital Front End (DFE).

Color quality counters

includes two methods for counting printed spreads. Each product bin will be configured to use one of the following counter types.

  • Counter per Press
  • All Presses

A product bin can be configured to track spreads in one of two ways; Counter per Press or All Presses. With either of these, the purpose of the counter is to track the number of printed spreads associated with a given product bin. The Counter per Press type is made up of the combination of a single product bin and a single press. This means that if you have a single product bin called "InfinityBooks", and this collection of products can be printed on any of the 5 presses within your environment, there will be 5 counters tracking the combination of the product bin and an individual press respectively.

With the All presses counter, every printed spread for the products defined within the product bin will result in the counter incrementing regardless of the press the job was submitted to.

The product bin counter the job was submitted to will increment by a value of 1 for each spread printed. It is important to understand that if the job does not show as printed within , the counter will not increment. This can occur if printed status cannot be obtained from the DFE. If this occurs, you can use the Mark Press Job as Printed option from the right-click menu on the press jobs tab to manually mark the job as printed until the print status issue has been resolved.

Resetting counters

Administrators can perform a reset on any of the counters defined in the system. This action will reset the value for current counter back to 0.

Reset counter
  1. Click the Reset Counters button on the Color Quality tab
  2. Select the desired counter from the drop down list
  3. Click the Clear button
  4. Click Ok on the confirmation dialogue
  5. Close the Clear Counters dialogue

Color Quality UI

The Color Quality tab provides the interface for working with and viewing information specific to color targets. From the Color Quality tab, the following actions can be performed.

  • Indicate start of shift
  • Generate color target
  • Modify configuration
  • Clear counters
  • View counters
  • View historical events

Indicate Start of Shift

This will set the start of shift that is used with the light volume threshold. Light volume targets will be discussed later in this chapter.

 Generate Color Target

The Generate Color Target button is used to manually submit a color target to a given press.

 Modify Configuration

Launches the web UI for configuring automated color targets

 Reset Counters

Provides a way for administrators to reset the value of current counter for one of the defined counters

 View Current Counters

Clicking this button will display a grid of all current counters.

 Color target history

The history section of the color quality tab displays information related to color target submission and start of shift. Some of the items that can be found in history include automatic vs. manual submissions, submission success and failure messages, start of shift time stamp, as well as several properties for each.

Administrative functions

Access to the admin tool and ability to reset counters can be restricted to certain users and / or groups using the access control facilities. The two access control tasks specific to the Color Quality system are qcsProductionAdmin and qcsConfigurationAdmin. The production admin controls access to the Reset Counters button and the configuration admin controls access to the Configure button used to modify the Color Quality configuration. The accesscontrol.xml file located in sharedconfig can be modified to restrict or allow specific groups to each task in order to control access as desired.
