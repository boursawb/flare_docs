Troubleshooting decorators

Issue Suggestion
New attribute not being applied to orders as expected

Verify the decorator is using the appropriate matcher.

Ensure the spelling and case of the matcher within the decorator is identical to the spelling and case of the matcher within the ordersort matcher folder.

Ensure only the name of the matcher was specified in the decorator, not the .matcher.xml portion of the name.

Confirm that the name of the matcher within the ordersort matchers folder includes the .matcher.xml suffix in the file name.

Verify the appropriate workflow and object context have been specified for the attribute that is being decorated.

Verify that the decorator is running at an appropriate workflow context to ensure the matcher will find the attributes required.
Unable to modify existing attributes

Verify that the data type specified in the decorator for "type=" matches the data type of the attribute being decorated. This information can be found on the view order attributes web page, or view order attributes from DB web page.

Verify the spelling and case of the attribute being modified by comparing what has been defined in the decorator to what displays on the view order attributes web page. It is always recommended to copy the attribute name from the view order attributes web page and paste it in to the appropriate field within the order decorator.
Orders are being decorated, but with an incorrect / unexpected value Verify there is only a single decorator attempting to set the attribute. In the event there are multiple decorators configured to set the same value for the same attribute, the one with the highest run level will take precedence.
Attribute not applied when specified in onNoMatch section of decorator

Ensure the decorator XML is well formed.

Verify from a working example that the onNoMatch section has been defined correctly within the decorator.

If attempting to decorate an existing order attribute, modify the attribute name to create a new attribute. This will verify that the original attribute value in question can be modified
