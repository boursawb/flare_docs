Auto-release configuration

Before setting up or modifying any auto-release criteria, it is recommended to enable the Sort Rule, Sort RuleGroup and Release Criteria columns on the sort groups to assist during the configuration process. The following flow shows an overview of the steps required for configuring auto-release within IPS.

The release criterion only specifies when the next batch is to be released, not how much work will be released. You can think of the release criteria as an operator standing by ready to select the "Release next batch" function.

We will use an example to illustrate releasing a batch when the total batches for a sort group exceeds 1.  This will prevent multiple batches of work sitting idle and will ensure that each batched released is full.  Whenever possible, it is recommended to use the order sort configuration utility to prevent errors that can occur when manually creating and editing XML files.  Please note that some manual XML editing will be required to complete the auto-release configuration process. 

Matcher attributes

Not all attributes you may wish to use when building matchers for auto-release will be available in the Order Batch Sort Utility. All of the attributes shown below, including any attributes you have added to the system roll-down configuration, can be used.

  • Estimated Pages
  • Estimated Sheets
  • Estimated Spreads
  • Estimated Waste Pages
  • Estimated Waste Percent
  • Estimated Waste Spreads
  • Manual Release Only
  • Next Batch Estimated Pages
  • Next Batch Estimated Spreads
  • Next Batch Estimated Waste Pages
  • Next Batch Estimated Waste Percent
  • Next Batch Estimated Waste Spreads
  • Remake
  • Sort Order
  • Sort Rule
  • Total Batches
  • Total Orders
  • Total Spreads All Buckets

This is not an exhaustive list and can grow as improvements are made to the software and custom attributes are added.

Configuration walk-through

  1. Launch the Order Sort Configuration utility
  2. Click the Create new matcher link in the Matcher actions section
  3. Specify a name of Total Batches
  4. In the Attribute drop-down, type in Total Batches (with the space between total and batches)
  5. Select the > symbol in the is drop-down
  6. Enter 1 in the value files
  7. Click ok to save your changes and create the new matcher and minimize / close the Order Sort Configuration utility
  8. Browse to \\<yourMasterCoreHostServer\idf\sharedconfig\ordersort\releasecriteria and edit any of the existing release criteria XML files showing in the folder
  9. Set the description= to Total Batches
  10. Set the matcher= to Total Batches
  11. Verify that autoBatchEnabled= is set to True
  12. Perform a Save As on the file and name it totalBatches.releasecriteria.xml
  13. Edit each sort group the new release criteria applies to and change the releaseCriteria= to Total Batches
  14. Return to the IPS application and go to the sort groups tab
  15. Scan the Sort Rule Group column for the groups that were modified and verify that the Release Criteria column shows the appropriate release criteria

If the release criteria column (possibly others) fails to display information, there is an error in one of the sort rule configuration files.  It is important to note that all attributes and file names in the sort rule configuration process are space / case sensitive.  A second indicator that an error exists within the sort rule configuration present as no batches displayed when expanding a sort group

If the configuration was successful, you will see the new release criteria show up in the Release Criteria column on the sort groups tab.
