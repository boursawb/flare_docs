Auto-release filters

Auto-release filters provide a way to scale back what is currently being automatically released from the system.  The walk-through demonstrates how to restrict the work being released to only those orders that have a priority code of H (high priority). 

Auto-release filter requirements

  • Auto-release is enabled
  • A valid release criteria has been defined
  • The release criteria in use has autoBatch=”true” defined
  • A valid auto-release filter has been defined
  • Auto-release has been configured to use a specific filter

Build the matcher and filter profile

  1. Open the Order Sort Configuration utility and create a new matcher called PriorityH
  2. Set the attribute to priorityCode
  3. Set is to ==
  4. Set to value to H
  5. Click ok to save the new matcher
  6. Minimize or close the Order Sort Configuration utility
  7. Browse to \\<yourServer>idf\sharedconfig\ordersort\autoreleaseprofile and open any one of the files displayed via an XML edit
  8. Set the description to Priority H
  9. Set the matcher to PriorityH
  10. Perform a save as and save the file with the name HighPriority.autoreleaseprofile.xml
  11. If Auto-release is currently active (with no filter applied), click the Turn off auto-release button on the sort groups tab
  12. Click the button again to enable Auto-release and specify an auto-release filter to limit the orders released based on the criteria defined above
  13. Click ok to apply the settings
