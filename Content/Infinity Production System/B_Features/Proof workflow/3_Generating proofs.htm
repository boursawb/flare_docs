Generating proofs

The following information describes how to mark and generate a proof batch from the products and sort groups tabs.

Marking an order as Proof Required (Products tab)

Any order within the system can be marked as Proof Required by simply right-clicking on the order and selecting the Mark Order Required Proof option from the Proofs sub-menu, as shown below.

  1. Highlight the order that requires the proof
  2. Right-click on the order
  3. Navigate to the Proofs menu
  4. Select the Mark Order Required Proof option

After a few seconds, the Proof Needed attribute will display yes in the UI as shown below.

Generating a proof from the Products tab

The following steps illustrate how to generate a proof batch from the products tab for an order that has been identified as Proof Required.

Generating a proof from the products tab
  1. Highlight the desired order that has been marked as Proof Needed / Yes
  2. Right-click on the order and select the Create Proof option from the Proofs menu
  3. Click Yes on the confirmation dialogue to generate the proof batch
  4. Select the appropriate destination device from the batch dialogue

This will generate a new proof batch and submit it to press with a priority of 100 and a quantity of 1.

Generating a proof from the Sort Groups tab

The following steps illustrate how to generate a proof batch from the Sort Groups tab for orders that have been identified as Proof Required.

Generating a proof from the sort groups tab
  1. Highlight the desired Proof Needed sort group
  2. Right-click on the sort group and select the Create Proof Batchoption from the menu
  3. Select the desired order that should be used to create the proof batch
  4. Select the appropriate destination device from the batch dialogue

Step number 3 from above will result in a dialogue where you will select the order ID to generate the proof batch for, as shown below.

This will generate a new proof batch and submit it to press with a priority of 100 and a quantity of 1.
