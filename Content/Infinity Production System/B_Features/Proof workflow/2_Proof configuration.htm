Configuring the environment

The primary configuration that should be performed to ensure the best overall experience when working with proofs is to include the proofNeeded attribute within the defined split by lists in the order sort configuration. Beyond that, there is no system level configuration required to use the feature. Building a UI perspective specific to creating a managing proofs will greatly improve the overall experience while using this feature.

Suggestions for Proofs UI persective

  • Enable the columns Proof, Needs Proof, Proof Sent Timestamp on the Products tab
  • Enable the columns Proof, Needs Proof, Proof Sent Timestamp on the Sort Groups tab
  • Enable the columns Proof, Needs Proof, Proof Sent Timestamp on the on the Batches tab
  • Enable the columns Proof, Needs Proof, Proof Sent Timestamp and Priority on the Press Jobs tab
  • Filter the Products, Sort Groups, Batches and Press Jobs tab by the Proof column being yes
  • Create a Quick Stat on the Products tab for Total Orders Requiring Proof
  • Create a Quick Stat on the Products tab for Total Proof Orders
  • Create a Quick Stat on the Products tab for Oldest Proof Order Timestamp

For additional information on working with UI Perspectives and Quick Stats, see "UI Perspectives and Quick Statistics" on page 1
