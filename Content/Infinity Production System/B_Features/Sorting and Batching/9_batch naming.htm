Modifying batch and press job names

allows you to define the attributes used that will make up the batch name, and at the same time, format the values as needed. Additionally, you can define the attributes that will be used for generating the press job names resulting from the generated batches. The configuration for press job naming can be done globally, or on a product-by-product basis for a given batch component within the product's batch definition.

Use custom batch names

  1. Browser to the sharedconfig folder for the environment requiring the change
  2. Locate and open the autobatch.config.xml file
  3. Include the attributes in the order to be displayed within the batchIDAttributes section of the configuration file, separated by commas
  4. Define any required string formatting within the batchIDFormatString section of the configuration file
  5. Save and close the autobatch.config.xml file
  6. Restart all DCore services

Custom press job naming

Similar to batch names, press job names can be customized as well. Press job naming can be configured globally using the naming strategy attributed defined within the batch.config.xml file, or for a given component within a batch definition by using an attribute of the same name. .

Use custom press job names
  1. Open the batch.config.xml file located in the sharedconfig folder
  2. Locate the defaultBatchComponentNamingStrategy section of the configuration file
  3. Include all of the attributes that will be used for the press job name within the defaultBatchComponentNamingStrategy tags
  4. Save and close the file

Job names can be as simple or complex as needed. Typically, each attribute will be separated by a hyphen or underscore and will include items such as the batch ID, batch component name, and product type. An example press job naming strategy is shown below.

Batch component level configuration

If you need to have certain products use a different naming strategy that has been defined for the system default, you can do this by adding the naming strategy attribute to the batch component for one or more products.

Component specific job naming

  1. Open the batch definition which contains the batch component that should use the custom naming strategy
  2. On the batch component line, add the attribute namingstrategy with the desired values.
  3. Save and close the file

When using a custom naming strategies, the press job name defined within the batch component section of the batch definition will override the settings defined at the global level. Also note that naming strategies defined at either level can use statically entered text and are not limited to batch attributes.
