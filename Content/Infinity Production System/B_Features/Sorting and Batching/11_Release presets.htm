Release Presets

If your system has been configured for auto-release, or you release batches based on the current device group configuration, it may be necessary to specify which components of a given product should be released to press. By default, every component defined within a product is submitted to press when using these release options. In a situation where one or more components should be excluded from the batch, a release preset can be defined to identify batch component exclusions, as well as production intent defaults. Release presets simply provide a way to tell the system which batch components to include during the batching process and provide a way to define a default intent when more than one intent has been built for a given component. has been designed to allow for multiple release preset profiles which can be selected as needed to accommodate changes in the production environment. The active release preset can be viewed on the sort groups tab. To change the active release preset, click the button to launch the Release Preset Web UI. Only one release preset profile can be active at any given time.

Understanding release presets

Before adding any presets to the Infinity system, it is important to have a basic understanding of the following key terms.

  • Release Preset This defines a single entry within the release preset configuration file. A Release Preset is specific to a single product type and will include one or more selections for the product
  • Product Type This defines a specific product within the system. A typical product might be an 8x10-Story-Book, or 3x5-Post-Card. Only a single product type can be included in a given preset
  • Selection One or more selections will exist within each release preset. A selection represents a single component within a product. The 8x10 book example, may have 3 components - book, cover, and wrap. Each of these components will result in their own selection within a release preset. Each selection will include the batch component name, the production intent to release, if defined, and a flag indicating if the selection should be released during the batching process.

As it is important to understand these terms and key concepts, much of this information is hidden from the administrator when using the IPS Production Admin web interface for configuring release presets. When there is a need to work with the XML configuration files directly, understanding how presets are defined and what each of these terms mean becomes more important.

Configuring release presets

Management of release presets is done through the IPS Production Admin web interface. The web UI can be accessed by clicking the button on the sort groups tab of the UI. From the home page, the following actions can be performed.

  • Select the active release preset default
  • Display and manage all presets
  • Modify component selections

Setting the active preset

  1. Expand the top level Release Presets menu if not already
  2. Click on Active Release Preset
  3. Click the EDIT button to the right of the active release preset
  4. Select the desired release preset from the drop-down list
  5. Click Save

Managing presets

  1. Click Release Presets in the menu
  2. Click the EDIT button to the right of the preset
  3. Make the necessary changes to the preset
  4. Click the Save button to commit any changes

The following actions are available within the Edit screen.

  • Provide a description for the preset
  • View/edit the release preset selections
  • Clone the release preset
  • Delete the preset

Modify component selections

The Component Selections page allows you to change the default intent for a product's batch component. From this screen you can perform the following actions.

  • Filter the list by selecting a specific release preset from the drop-down menu
  • Filter the list by specifying a key word in the search field
  • Export the list of release presets to a .csv formatted file
  • Edit a product to change or exclude a particular production intent from a component

To change or exclude a production intent from a batch component, follow the steps shown below.

  1. Click the EDIT to the right of the batch component
  2. Select the appropriate production intent from the drop down list, or alternatively click the X to exclude the production intent
  3. Click Save
