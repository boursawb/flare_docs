Presort

Presort is an operation performed on an order before reaching the final sort phase of the workflow. During presort, both sort rules and matchers will be honored, as well as the ability to match on a mix of order and product level attributes. The result of presort is a set of new attributes on the order that can be used by subsequent steps in the workflow or by order decorators. Presort can occur at the following three phases of the order workflow.

  • Inject
  • Analyze
  • Prebatch

Each presort phase will result in 4 new order attributes with a default value of wildcard. The four attributes generated for the analyze presort phase are shown below.

  • analyzePreSort Id
  • analyzePreSort Rule
  • analyzePreSort Rule Group
  • analyzePresortRule

A value of wildcard indicates that either no analyze presort rules yet exist, or the order did not match any of the defined analyze presort rules. A very basic rule of thumb for determining if presort will be required is to ask the following question.

Do I need to identify orders based on the values of both order and product level attributes?

If the answer to this question is yes, then presort will likely be required.
