Manual device selection

offers two methods for submitting work to press, auto device selection and manual device selection. Manual device selection allows you to choose a specific destination for the job to be sent. Automatic device selection relies on predefined device groups which determine the device the batch will be submitted to.

Manual device selection occurs when a Create Batch action is initiated from the Products tab or when a manual release/reprint option is selected from the sort groups, batches or press jobs tabs. All manual release options will result in a dialogue box displaying which contains the destination(s) that each component can be sent to. Additionally, you will have the ability to turn off any components which may not need to be printed. The six options for manually releasing work to press are shown below.

  • Create Batch - Products tab
  • Create Single Product Batch - Products tab
  • Manual - Release Next Batch - Sort Groups tab
  • Manual - Release All - Sort Groups tab
  • Release Many - Manual - Sort Groups tab
  • Manual Reprint Batch - Batches/Press Jobs tabs

Manual device restrictions

Administrators have the ability to restrict the devices available to a given product during a manual batch process. Device restrictions can be applied on a product by product basis, on all products associated with a given sort group, or for all products defined for a given destination type. When a manual batch creation is performed, and the product contained within the batch uses a device group containing manual device restrictions, the batching process will be restricted to only those devices that the administrator has allowed. Manual device restrictions are configured within the device group associated with the product, sort group or destination type requiring them.

Setting up manual device restrictions

  1. Identify the device group that is used during a normal release
  2. Add a manual device section to the device group configuration file as shown below
<devicegroup desciption="A test group" deviceType="indigo" >
	<prioritydevices loadbalance="random">
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Prio-1">
		</device>
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Prio-2">
		</device>
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Prio-3">
		</device>
	</prioritydevices >
	<remakedevices loadbalance="random">
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Remake-1">
		</device>
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Remake-2">
		</device>
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Remake-3">
		</device>
	</remakedevices>
	<standarddevices loadbalance="random">
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Standard-1">
		</device>
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Standard-2">
		</device>
		<device dfe="ULTRA-3-6-1" device="Cal-DeviceGroup-Standard-3">
		</device>
	</standarddevices>
	<manualdevices>
		<device dfe="ULTRA-3-6-1" device="Cal-Manual-Standard-1">
		</device>
	</manualdevices>
</devicegroup>
  1. Include the DFE / device pair entries that will make up the manual devices
  2. Save the device group configuration file

Manual device restrictions provide a way to ensure that only the appropriate / desired choices are selected when batching work to press.
